### Production-code fix surfaced by integration tests

`scripts/rename-hook.sh` line 154 had `LLM_OUTPUT=$(cmd || echo '{...}')`
which concatenated stdout from both arms when `cmd` failed AND already
emitted `{"error":"call_failed"}` on its own stdout. The duplicated
output broke `jq --argjson o "$LLM_OUTPUT"` in the subsequent
`log_event llm_call_end` call (warning: "invalid JSON text passed to
--argjson"). Commit 7ed2317 replaces the pattern with an unconditional
call + empty-fallback.
